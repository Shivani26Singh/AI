# LLM.md — STLC Agent Project Constitution

## Data Schema

### Input
```json
{
  "prd_pdf": {
    "fileName": "string",
    "mimeType": "application/pdf",
    "fileSize": "number",
    "binaryData": "binary"
  }
}
```

### Intermediate: Extracted PDF Text
```json
{
  "text": "string (full PRD text)",
  "fileName": "string"
}
```

### Output: Test Plan (Markdown)
Generated by "Generate Test Plan" AI Agent → Saved as **Test_Plan.md**

### Output: Test Cases (CSV)
Columns: Requirement ID, Acceptance Criteria ID, Scenario, TID, Category, Priority, Test Data, Test Case Description, Pre-Condition, Test Steps, Expected Result, Actual Result, Status, Executed By, Comments, Automation Candidate
Generated by "Generate Test Cases" AI Agent → Saved as **Test_Cases.csv**

### Output: Playwright Tests (TypeScript)
Generated by "Generate Playwright Tests" AI Agent → Saved as **playwright_tests.ts**

## LLM Configuration
- Provider: Groq
- Model: `llama-3.3-70b-versatile` (shared across all 3 AI Agents)
- nodeType: `@n8n/n8n-nodes-langchain.lmChatGroq` (v1)
- All 3 agents connected in single `ai_languageModel` array

## Behavioral Rules
- Do NOT invent features. Derive all coverage from PRD content only.
- Use placeholders for missing information.
- Output ONLY the requested format (markdown for TP, CSV for TC, TypeScript for PW).
- No AI disclaimers, explanations, or commentary in output.

## Architectural Invariants
- 12-node workflow chain with fork pattern
- Form (v2.5) → Extract (v1.1) → TP Gen (v3.1) → [Save TP (v1.1) + Wait → TC Gen (v3.1)]
  → [Save TC (v1.1) + Wait → PW Gen (v3.1)] → Save PW (v1.1)
  → Merge (v3.2) → Combine Code (v2) → Download Form (v2.5)
- Single Groq LLM shared across all 3 AI Agent nodes
- System prompts embedded inline (not separate Skill nodes)
- Wait nodes (2s) between consecutive AI Agent calls for rate-limit protection
- ConvertToFile uses `operation: "toText"` and `sourceProperty: "output"`
- AI Agent prompts use pattern: `=Generate...\n\n{{ expression }}`
- Fork pattern preserves text flow: each agent's text output feeds the next agent
